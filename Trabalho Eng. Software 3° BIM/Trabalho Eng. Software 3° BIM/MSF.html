MSF - MICROSOFT SOLUTIONS FRAMEWORK

O MSF surgiu a partir da análise de como a Microsoft desenvolve seus produtos. O MSF é uma compilação de boas práticas utilizadas pela empresa, que foi criado tanto para uso interno como para uso de seus clientes.
Apesar de ter sido criada pela Microsoft, o MSF aborda basicamente o processo de construção de soluções, não se prendendo ao uso de produtos desta empresa.
A Microsoft não classsifica o MSF como metodologia, mas sim como uma disciplina. Basicamente o MSF serve como um grande guia e uma coleção de boas práticas.
O MSF não se aprofunda em detalhes. A falta delas pode parecer uma deficiência, mas essa característica permitiu uma abordagem simples e direta das técnicas apresentadas. Ou seja, o MSF permite uma fácil compreensão, além de ser flexível.
Os modelos de equipes são compostos por 6 papéis e se baseia no conceito de "Team of Peers", ou seja, todos os papéis são iguais, sem hierarquia.

Modelo de equipes:

Product Management: Representar o cliente para a equipe;
Program Management: Gerenciar o andamento do projeto e atuar como facilitador;
Development: Projeto e construção da solução;
Testing: Testar a solução , abordar os problemas antes da entrega;
User Education: Treinar o usuário e otimizar a performance;
Logistics Management: Planejar e executar a logística para uma boa implantação e manutenção;

O prícipio deste modelo é que cada um desses papéis aborda um objetivo importante para o projeto.
Fica claro que a boa comunicação permite rápida tomada de decisões sobre o projeto, levando em consideração todos os obejtivos do mesmo.

Modelos de Processos:

O modelo preve 4 fases: Envisioning, Planning, Developing e Stabilizing;

O Envisioning tem como produto principal um documento de visão e escopo. Este documento formaliza de forma suscinta a visão do projeto.
O Planning, tem como produto o plano do projeto, que é composto de subprodutos, dos quais destamos a especificação funcional e o cronograma do desenvolvimento.
O Developing, é quando construímos a solução. Acaba quando todas as funcionalidades estejam funcionando.
O Stabilizing é dedicada a testes de sistema e correções de bugs.
